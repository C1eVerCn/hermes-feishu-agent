      把示例 open_id 换成**你自己真实的飞书 open_id**,并准备三个不同角色账号:

- 一直「非平台用户」→ identity_map 的 open_id 是否是你真实 open_id、email 是否在 fake_db 中存在
- Agent 编造邮箱/台架编号 → 检查系统提示是否生效、`list_available_benches` 是否被先调用
